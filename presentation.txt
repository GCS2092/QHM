# 📘 CAHIER DES CHARGES COMPLET

## Application Web d’Évaluation, Scoring et Analyse (Domaine Fiscalité)

---

# 1. Présentation du projet

## 1.1 Nom du projet

**FiscalScore – Plateforme d’évaluation et d’analyse comportementale client**

---

## 1.2 Contexte

Dans un contexte de **fiscalité et gestion de clients**, l’entreprise souhaite disposer d’une application web permettant :

* d’évaluer des clients selon des critères définis (comportement fiscal, conformité, régularité),
* de collecter des données via questionnaires,
* de calculer un score global,
* d’interpréter ce score selon des seuils,
* de générer une fiche client exploitable,
* et de contrôler strictement les accès aux données sensibles.

---

## 1.3 Objectif principal

Créer une plateforme permettant de :

* transformer des réponses en **score fiscal/comportemental**
* classer les clients selon leur niveau de risque ou conformité
* produire des rapports exploitables par les agents

---

# 2. Utilisateurs du système

| Rôle                   | Description                           |
| ---------------------- | ------------------------------------- |
| Administrateur         | Accès total + gestion système         |
| Agent / Évaluateur     | Remplit les questionnaires            |
| Superviseur            | Consulte les résultats                |
| Utilisateur standard   | Accès limité à ses propres données    |
| Accès temporaire admin | Déverrouillage via mot de passe admin |

---

# 3. Fonctionnalités principales

---

# 3.1 Gestion des clients (personnes évaluées)

Chaque client possède une fiche :

## Données obligatoires

* Nom
* Prénom
* Identifiant fiscal (important pour ton cas)
* Téléphone
* Email
* Adresse
* Statut fiscal (optionnel)
* Historique des évaluations

---

## Actions possibles

* créer un client
* modifier
* supprimer (admin uniquement)
* rechercher
* consulter historique

---

# 3.2 Gestion des questionnaires

Les questionnaires sont liés à la fiscalité :

## Exemples de thèmes

* conformité fiscale
* ponctualité des déclarations
* historique de paiement
* transparence financière
* niveau de risque fiscal

---

## Chaque questionnaire contient :

* questions
* catégories
* coefficients
* barèmes de notation

---

# 3.3 Gestion des questions

## Types possibles :

* choix unique
* échelle (1 à 5)
* texte libre (commentaire)
* score automatique

---

## Exemple

> Le client déclare-t-il ses revenus dans les délais ?

| Réponse  | Score |
| -------- | ----- |
| Toujours | 5     |
| Souvent  | 4     |
| Parfois  | 2     |
| Jamais   | 0     |

---

# 3.4 Commentaires par question

Chaque réponse peut contenir :

* un commentaire de l’évaluateur
* une justification

---

# 3.5 Calcul des scores

## Formule officielle

Score = \frac{\sum (note \times coefficient)}{\sum coefficients}

---

## Résultat final :

* Score sur 100
* Classification automatique

---

# 3.6 Interprétation des scores

| Score    | Profil client fiscal  |
| -------- | --------------------- |
| 0 – 20   | Client à risque élevé |
| 21 – 40  | Risque modéré         |
| 41 – 60  | Conforme moyen        |
| 61 – 80  | Bon profil fiscal     |
| 81 – 100 | Excellent / fiable    |

---

# 3.7 Génération de fiche client

## Contenu PDF :

* identité client
* score fiscal
* analyse comportementale
* commentaires
* historique
* recommandations

---

# 3.8 Dashboard

* nombre de clients
* clients à risque
* évolution des scores
* statistiques globales
* filtres avancés

# 4.1 Rôles standards

| ----------- | ------- |
| Admin       | total   |
| Agent       | limité  |

---

# 4.2 Accès avancé par mot de passe admin

## Principe

Un utilisateur peut accéder à des données normalement interdites **si et seulement si** :

* il saisit le mot de passe administrateur,
* ou obtient validation admin.

---

## Exemple

```text id="access_flow_1"
Utilisateur clique module sécurisé
        ↓
Popup "mot de passe admin requis"
        ↓
Validation correcte
        ↓
Accès temporaire activé
```

---

# 4.3 Accès temporaire

* durée limitée (ex: 10 minutes)
* ou session active uniquement

---

# 4.4 Journalisation obligatoire

Chaque accès admin doit être tracé :

| champ           |
| --------------- |
| utilisateur     |
| action          |
| heure           |
| IP              |
| module consulté |

* ou 2FA (code SMS / email)
* ou token temporaire
